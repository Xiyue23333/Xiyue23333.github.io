CHAPTER 01 · SECTION 02

第二节：学会查看官方代码

状态：完整教程 示例来源：E:\官方文件\client-extra

本节目标

  • 知道“官方文件”都放在哪（客户端 jar / client-extra / data pack / assets）。
  • 学会在 IDEA 里快速定位：从你写的代码跳到官方类、搜索资源文件、对照 key/路径。
  • 掌握最常用的入口与阅读路线：从“你要做的效果”反推到“官方是怎么做的”。

重要提醒：Minecraft 官方文件（含资源与数据）受版权保护，不建议把完整内容上传到公开仓库或 GitHub Pages。本页提供方法与“本地生成”方案，用于你自己电脑离线学习。

官方文件在哪？（以 1.20.1 为例）

你电脑里的示例路径是 E:\官方文件\client-extra。它主要包含：

  • assets/：客户端资源（语言文件、纹理、着色器、字体、音效等）。
  • data/：数据驱动内容（世界生成、结构、标签、战利品表等）。
  • version.json：版本相关信息。

在模组开发里，你最常“抄作业”的不是 Java 源码，而是这些 资源/数据文件的结构与写法：路径怎么命名、key 怎么写、json 字段代表什么。

在 IDEA 里怎么查？（最快用法）

  1. 从你写的代码跳到官方类：对着类名/方法名按 Ctrl + B（或 Ctrl + Click）。
  2. 全局搜索：Ctrl + Shift + F 搜索字符串（比如一个翻译 key、一个资源路径、一个注册名）。
  3. 按类名搜：Ctrl + N 搜类；按文件名搜 Ctrl + Shift + N（找 json/纹理/着色器很常用）。
  4. 看资源路径：记住两大根目录：assets/<命名空间>/ 与 data/<命名空间>/。

建议的练习方式：先定一个目标（例如“我要加一个语言翻译 key”或“我要看世界生成 noise_settings 怎么写”），然后用搜索把官方写法找出来，对照着写。

本地 IDE 工程视图（左选文件，右看内容）

下面这个窗口默认展示的是“示例/占位文件”。如果你在本机运行生成脚本，它会把 E:\官方文件\client-extra 的选定文件嵌入到本页的 .local.html 版本里，达到和第一节一样的离线阅读体验。

  1. 在站点根目录运行：powershell -ExecutionPolicy Bypass -File scripts\generate-client-extra-local.ps1
  2. 生成文件：guides/ch1-sec2-read-official-code.local.html
  3. 用浏览器打开该 .local.html 页面即可离线查看（不要上传到公开仓库）。

阅读路线建议（照着练）

  • 想改显示文本：先找 assets/*/lang/*.json，学会 key 命名与复用。
  • 想抄一个 UI 贴图/布局：先找 assets/minecraft/textures/ 与相关 json（模型/字体/GUI）。
  • 想做世界生成：先找 data/minecraft/worldgen/，从 noise_settings、density_function 开始看。
  • 想知道某个行为怎么触发：在你写的事件/物品/方块代码里按 Ctrl + B 跳到被调用处，再向上追调用链。
