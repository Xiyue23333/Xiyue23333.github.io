CHAPTER 01 · SECTION 02

第二节：学会查看官方代码

状态：完整教程 主题：IDEA 外部库中的官方文件

本节目标

  • 知道官方“数据文件”和“源代码”在 IDEA 的哪个位置。
  • 学会在开发过程中随时对照官方实现，辅助你写自己的 Mod。

第一步：找到“外部库”（External Libraries）

打开你的 Mod 工程后，在 IDEA 左侧项目窗口中，拉到最下方可以看到 External Libraries（外部库）。

外部库里包含了你工程依赖的各种库，其中就包括 Minecraft 官方文件。

提示：如果你看不到外部库或里面内容不全，先在 IDEA 右侧 Gradle 面板点击刷新/重载，或回到第一节提到的“重构 Gradle”。

第二步：官方“数据文件库”（client-extra）

在 External Libraries 中找到：

Gradle:net.minecraft:client:extra:1.20.1

这就是官方的 数据文件库（可以理解为官方资源/数据文件的集合）。

  • 在该库的 assets 目录里，可以找到官方的贴图、模型等资源文件。
  • 在该库的 data 目录里，可以查看掉落物、配方等数据文件（以及更多数据驱动内容）。

第三步：官方“源代码库”（Forge 映射后的源码）

继续往下找 External Libraries 中的：

Gradle:net.minecraftforge:forge:1.20.1-47.4.10_mapped_parchment_2023.09.03-1.20.1

这是官方的 源代码文件库（Forge + 映射后，你能更容易读懂类名与参数）。

主要内容在该库下的 net\minecraft 目录中，你可以在这里查看：

  • 官方物品、方块、实体等是如何实现的。
  • 各种机制（交互、事件、渲染、网络、数据同步等）在官方里怎么写。

怎么用它来辅助写代码？

  • 当你不知道“应该怎么写”时，先在 net\minecraft 里找一个最接近的官方实现做参考。
  • 当你不知道“资源/数据文件怎么写”时，去 net.minecraft:client:extra 里对照官方的 assets/data 结构与字段。
  • 后面编写代码时，多查官方文件，可以显著减少踩坑。

快捷跳转（建议记住）

  1. 跳转到定义：Ctrl + B / Ctrl + Click
  2. 全局搜索：Ctrl + Shift + F
  3. 按类名搜：Ctrl + N
  4. 按文件名搜：Ctrl + Shift + N
