CHAPTER 01 · SECTION 03 · POINT 04

第三节·第四点：方块掉落物

状态：完整教程 来源：E:\教程mod

本节目标

  • 让矿石方块 crystal_ore 在被挖掘时掉落 crystal，而不是总是掉自己。
  • 让矿石只能被镐子正确挖掘，并要求最低工具等级为石镐。
  • 理解 Java 方块属性、掉落表、方块标签三者分别负责什么。

先理解：掉落物和挖掘等级不是一回事

很多初学者第一次做矿石时，会把“掉什么物品”和“什么工具能挖”写混。实际上这两件事分属不同位置：

  • ModBlocks.java 中的 .requiresCorrectToolForDrops()：告诉游戏“这类方块只有用正确工具时才应该正常掉落”。
  • data/minecraft/tags/blocks/mineable/pickaxe.json：告诉游戏“这个方块属于镐子挖掘”。
  • data/minecraft/tags/blocks/needs_stone_tool.json：告诉游戏“这个方块最低需要石镐级别”。
  • data/tutorial/loot_tables/blocks/crystal_ore.json：真正定义“挖掉后掉什么”。

注意：这四部分缺任何一个，效果都可能和你预期不一致。

第一步：在矿石方块属性里开启“正确工具掉落”

先在 ModBlocks 里找到矿石注册代码，给 CRYSTAL_ORE 的属性链加上：

.requiresCorrectToolForDrops()

它的作用不是指定“石镐”，而是告诉 Minecraft：这个方块必须结合工具标签来判断是否允许正常掉落。

如果你只写标签但不加这句，低级工具也可能照样掉落；如果只加这句但标签没配齐，就会出现“所有镐子都不掉落”。

第二步：创建矿石掉落表 crystal_ore.json

掉落表文件路径为：

src/main/resources/data/tutorial/loot_tables/blocks/crystal_ore.json

本节逻辑采用常见矿石写法：

  • 如果工具带有精准采集（Silk Touch），掉落矿石方块本身：tutorial:crystal_ore
  • 如果没有精准采集，掉落物品：tutorial:crystal
  • 掉落数量使用随机范围，并支持时运附魔加成

命名提醒：掉落表文件名必须与方块注册名一致。你的方块注册名是 crystal_ore，文件就必须叫 crystal_ore.json，不能写成别的名字。

第三步：告诉游戏“这是镐子挖的矿石”

在原版标签目录中创建：

src/main/resources/data/minecraft/tags/blocks/mineable/pickaxe.json

把 tutorial:crystal_ore 放进去，表示这个方块属于“镐子挖掘”的分类。

这一步只是在说“用什么工具类型”，还没有限制等级，所以单独写这个文件时，木镐也可能被允许正常掉落。

第四步：再加最低工具等级限制

接着创建：

src/main/resources/data/minecraft/tags/blocks/needs_stone_tool.json

同样把 tutorial:crystal_ore 放进去。这样 Minecraft 就会知道：

  • 这个方块要用镐挖
  • 而且至少要石镐级别

最终效果就是：木镐可以尝试挖，但不会正确掉落；石镐、铁镐、钻石镐及以上才能正常获得掉落物。

常见错误排查

  • 掉落表文件名和方块注册名不一致，例如方块是 crystal_ore，文件却写成别的名字。
  • pickaxe.json 没建，导致游戏不知道它属于镐子挖掘。
  • needs_stone_tool.json 路径写错。这个文件不在 mineable 目录下，而是在 tags/blocks 根目录下。
  • 忘记在 Java 里加 .requiresCorrectToolForDrops()，结果木镐也能掉落。
  • JSON 里写错资源名，例如误写成没有注册过的 tutorial:magic_crystal。

进游戏验证

  • 木镐挖 crystal_ore：方块会被破坏，但不应正常掉落 crystal。
  • 石镐及以上挖 crystal_ore：应掉落 crystal，并受时运影响。
  • 带精准采集的石镐及以上：应直接掉落 crystal_ore 方块本身。
