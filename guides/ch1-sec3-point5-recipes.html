CHAPTER 01 · SECTION 03 · POINT 05

第三节·第五点：物品的配方制作

状态：完整教程 来源：E:\教程mod

本节目标

  • 学会用原始 JSON 方式为模组添加工作台配方，不使用数据生成。
  • 为 crystal_block 制作正向配方：9 个 crystal 合成 1 个方块。
  • 为 crystal_block 制作逆向配方：1 个方块拆回 9 个 crystal。

为什么先学原始配方写法

虽然后面可以用数据生成自动产出配方，但在教程的现阶段，先手写 JSON 更容易理解。因为你能直接看到：

  • 配方文件应该放在哪个目录
  • 有序配方和无序配方的 JSON 结构分别长什么样
  • 结果物品、材料物品、数量是如何写进文件的

建议：先把手写配方学明白，后面再学 datagen，思路会更清楚。

第一步：创建配方目录

所有工作台配方文件都放在：

src/main/resources/data/tutorial/recipes/

这里的 tutorial 必须和你的模组 id 保持一致。也就是说，前面主类里写的是：

public static final String MODID = "tutorial";

那么配方目录就必须是 data/tutorial/recipes，不能写成其他命名空间。

第二步：正向配方，9 个水晶合成 1 个水晶块

矿石块、金属块、宝石块这类“9 合 1”的配方通常写成 有序配方（Shaped Recipe）。

创建文件：

src/main/resources/data/tutorial/recipes/crystal_block.json

然后用 3 行 3 列的图案来表示工作台摆放：

  • pattern：工作台布局
  • key：把图案中的字母映射到实际物品
  • result：合成结果

由于每一格都是水晶，所以最适合用同一个字符，例如 X。

第三步：逆向配方，1 个水晶块拆成 9 个水晶

反向拆解通常写成 无序配方（Shapeless Recipe），因为它不关心工作台格子的摆放顺序。

创建文件：

src/main/resources/data/tutorial/recipes/crystal_from_block.json

这类配方最适合写成：

  • ingredients：只写需要的材料列表
  • result.count：直接把数量写成 9

这样玩家只要把一个 crystal_block 放进工作台任意一格，就能拆回 9 个 crystal。

命名与内容要保持完全对应

这一步最容易出错的是资源名不统一。你当前模组里真正注册的是：

  • 物品：tutorial:crystal
  • 方块物品：tutorial:crystal_block

所以配方 JSON 里也必须写这两个名字，不能随手写成别的内部名。

注意：如果配方里写了未注册的物品 id，游戏加载数据包时就会报错，或者配方根本不会出现。

进游戏验证

  • 在工作台中摆放 9 个 crystal，应能合成 1 个 crystal_block。
  • 把 1 个 crystal_block 放入工作台任意格，应能分解为 9 个 crystal。
  • 如果配方不生效，优先检查文件路径、命名空间和物品 id 是否完全一致。
