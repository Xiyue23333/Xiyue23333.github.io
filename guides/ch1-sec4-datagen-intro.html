CHAPTER 01 · SECTION 04

第四节：优化代码（数据生成）[可选学（维新派）]

状态：完整教程（可选） 目标：从手写 JSON 迁移到自动生成

本节定位

前三节我们用手写 JSON 完成了资源流程，优点是直观、上手快。第四节开始引入 Data Generation（数据生成），把配方、标签、掉落表、模型、语言文件改为“由 Java 代码统一生成”，减少重复劳动并降低改名时的维护成本。

说明：本节是“工程化优化”，不是“入门必修”。你可以先把前三节做熟，再回来升级。

你会得到什么

  • 统一的资源生成入口，不再手动维护大量 JSON。
  • 新增内容时可以“一次改代码，多文件同步产出”。
  • 更清晰的模块拆分：配方、标签、掉落表、模型、语言各自独立 Provider。

推荐目录策略

保留这三层分工，最不容易乱：

  • src/main/resources：手写的核心资源（保底可运行）。
  • src/generated/resources：数据生成产物（由任务覆盖更新）。
  • src/main/java/.../datagen：各类 Provider 与入口类。

并确保 build.gradle 已包含：

sourceSets.main.resources { srcDir 'src/generated/resources' }

第一步：注册数据生成入口

在主类中监听 GatherDataEvent，把各 Provider 挂进去。示意结构如下：

@Mod.EventBusSubscriber(modid = TutorialMod.MODID, bus = Mod.EventBusSubscriber.Bus.MOD)
public final class DataGenerators {
    @SubscribeEvent
    public static void gatherData(GatherDataEvent event) {
        DataGenerator generator = event.getGenerator();
        PackOutput output = generator.getPackOutput();
        ExistingFileHelper helper = event.getExistingFileHelper();

        generator.addProvider(event.includeServer(), new ModRecipeProvider(output));
        generator.addProvider(event.includeServer(), new ModBlockTagProvider(output, event.getLookupProvider(), helper));
        generator.addProvider(event.includeServer(), new ModLootTableProvider(output));

        generator.addProvider(event.includeClient(), new ModBlockStateProvider(output, helper));
        generator.addProvider(event.includeClient(), new ModItemModelProvider(output, helper));
        generator.addProvider(event.includeClient(), new ModLanguageProvider(output, "zh_cn"));
    }
}

第二步：按功能拆 Provider

建议最少拆成 5 组：

  • 配方：ModRecipeProvider（如 crystal_block 与 crystal_from_block）。
  • 方块标签：ModBlockTagProvider（如 mineable/pickaxe、needs_stone_tool）。
  • 掉落表：ModLootTableProvider（矿石精准采集/时运逻辑统一放这里）。
  • 模型与方块状态：ModBlockStateProvider + ModItemModelProvider。
  • 语言：ModLanguageProvider（en_us / zh_cn）。

规则是：一个 Provider 只负责一类资源，后续查错和扩展都更快。

第三步：执行数据生成

使用 Gradle 的数据生成任务输出文件：

./gradlew runData

执行后重点检查：

  • src/generated/resources/data/tutorial/recipes
  • src/generated/resources/data/tutorial/loot_tables
  • src/generated/resources/data/minecraft/tags/blocks
  • src/generated/resources/assets/tutorial

从现有工程迁移的建议顺序

  1. 先迁移“配方 + 标签”（最简单、最稳定）。
  2. 再迁移“掉落表”（逻辑较多，验证成本高）。
  3. 最后迁移“模型/语言”（数量多但逻辑简单）。
  4. 每迁移一类就跑一次 runData + runClient，不要一次改完再排错。

常见问题

  • 生成了但游戏没读到：确认 src/generated/resources 已加入 main resources。
  • 命名空间错乱：所有资源都要和 MODID=tutorial 对齐。
  • 数据任务报版本问题：Forge 1.20.1 工程建议使用 JDK 17 运行 Gradle。
