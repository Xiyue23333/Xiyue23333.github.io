技术栈选择

不同 Loader 适用场景不一样。你可以先选一个主线，后续再做多平台适配。

Forge / NeoForge

生态成熟，教程多，适合从零入门和复杂内容型模组。

  • 优势：资料丰富、事件系统完整、兼容老项目迁移。
  • 建议：先做 1 个小功能模组，再扩展到 GUI、方块实体等。

Fabric

轻量灵活，版本更新快，适合快速迭代与工具型模组。

  • 优势：启动快、API 轻、社区工具链活跃。
  • 建议：搭配 Yarn 映射与 Loom，保持依赖版本一致。

常见问题

Q1: Java 基础很弱，能学模组开发吗？

可以。先掌握类、方法、集合、面向对象，再按本教程做小步实战。

Q2: 先学 Forge 还是 Fabric？

如果你是第一次做模组，优先 Forge/NeoForge；追求轻量和新版本可优先 Fabric。

Q3: 为什么进游戏后看不到我注册的物品？

先检查注册事件是否触发、命名空间是否一致、模型 JSON 与贴图路径是否匹配。
